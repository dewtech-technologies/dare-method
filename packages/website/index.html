The Method

What is DARE?

DARE is a 4-phase methodology that keeps humans thinking strategy and AI executing tactics. Every phase transition requires an explicit human checkpoint — no more runaway AI sessions.

Phase 01
DESIGN
Define what to build and why. Human-led with AI assistance. Output: DESIGN.md.
→
Phase 02
ARCHITECT
AI proposes architecture + task breakdown. Human validates. Output: BLUEPRINT.md.
→
Phase 03
REVIEW
Human checkpoint: approve or refine the plan before a single token of implementation is spent.
→
Phase 04
EXECUTE
AI implements task-by-task with the Ralph Loop — auto-correcting until validation gates pass.

CLI

One command to start

The DARE CLI scaffolds projects, manages skills, and runs the Ralph Loop — all from your terminal.

dare — zsh
$ npm install -g @dewtech/dare-cli
+ @dewtech/dare-cli@3.0.0
$ dare new myapp --stack rails
Scaffolding Rails 8 project with DARE structure… ✓ DARE/DESIGN.md — ready for your requirements ✓ DARE/BLUEPRINT.md — AI will fill this next ✓ .dare/config.json — project configuration
$ dare skill add dare-llm-integration
✓ dare-llm-integration@1.2.0 installed
$ dare skill list
dare-ax 1.4.0 ✓ active dare-layered-design 2.1.0 ✓ active dare-llm-integration 1.2.0 ✓ active dare-frontend-design 1.0.3 ✓ active dare-realtime 0.9.1 ✓ active dare-quality-telemetry 1.1.0 ✓ active
